# Compliance Boundaries

This repo is for production browser automation infrastructure. It is not a guide to abusing platforms or bypassing protected access.

Do not use these examples for:

- credential harvesting
- account automation
- scraping private or paywalled data without permission
- bypassing access controls
- ignoring applicable site terms, laws, or rate limits

Provider services can improve reliability, routing, and execution ergonomics. They do not grant permission to access data you are not allowed to access.

Before adapting an example:

- confirm the data is public or you have permission to access it
- review applicable site terms and robots guidance
- use rate limits and retry budgets
- store only data you actually need
- keep artifacts secure when they may contain customer or business data
